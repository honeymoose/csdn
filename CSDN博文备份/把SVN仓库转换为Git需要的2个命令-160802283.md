<p>将 SVN 转换为 Git 仓库的步骤是分多步进行的。</p> <br><p>主要流程是首先获得提交人的基本信息，然后根据提交信息修改提交人的配置文件。</p> <br><p>随后通过配置文件的映射来一步一步的将所有仓库中的原始提交数据转换为 Git 的提交数据。</p> <br><p></p> <br><pre><code>git svn clone --stdlayout --authors-file=authors.txt https://svn.code.sf.net/p/hawebs/svn git-migration<br>svn log -q https://svn.code.sf.net/p/hawebs/svn | grep -e "^r" | awk '{print $3}' | sort | uniq > authors.txt<br></code></pre> <br><p>通过上面的两个步骤，就可以将一个SVN仓库转换为 Git 的仓库，当然在转换的过程中，我们可能后续还需要将 Tag 和分支一并进行转换。</p> <br><p>但 Tag 和分支的转换，可能对一些小的项目来说并没有必要，因为 Tag 和分支并不多。</p> <br><p></p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/0/a/0a6756d80037b063dd19f6e708862fe5c4e15221.jpeg" rel="nofollow"><img alt="2026-05-0415-00-43" height="345" src="https://i-blog.csdnimg.cn/img_convert/b28c4838dbc48ce94ffb3af2907233e6.jpeg" width="690" /></a></p> <br><p></p> <br><p>在仓库的转换过程中，如果能够看到上图显示的输出信息，那么说明仓库目前正在转换过程中，基于仓库的大小，转换的时间可能会很长。</p> <br><p>因为这个工具需要逐条读取提交记录，然后获取提交信息，并且将提交信息转换为 Git 的提交历史记录。</p> <br><p>简单的来说，就是 Git 的转换工具将会逐条读取提交记录，逐条进行转换。</p> <br><p>因此如果你的提交记录很多，并且内容很复杂的话，那么转换的时间会很长，好在是转换界面中能够提看到转换的进度。</p> <br><p>上图显示的 R+加数字 就是当前处理的提交版本号。</p> <br><p></p> <br><p><a href="https://www.isharkfly.com/t/svn-git-2/10010" title="https://www.isharkfly.com/t/svn-git-2/10010" rel="nofollow">https://www.isharkfly.com/t/svn-git-2/10010</a></p>
