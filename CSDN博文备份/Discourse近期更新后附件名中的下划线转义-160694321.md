<p>最近在使用 Discourse 上传文件的时候，发现如果文件名中有一个下划线的话，文件名会被Discourse 进行自动转义。</p> <br><p></p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/9/5/952c85adb2951c8d2eb36246586c8509b4aa559f.jpeg" rel="nofollow"><img alt="2026-05-0110-00-53" height="341" src="https://i-blog.csdnimg.cn/img_convert/7530541dd5b888b1f31d121723d444ae.jpeg" width="690" /></a></p> <br><p></p> <br><p></p> <br><p>看上图文件名中的第一个下划线会被自动加上转义符。</p> <br><p>我觉得以前是没有这个现象的，官方针对这个现象的说法是： <a href="https://meta.discourse.org/t/incorrect-attachment-markup-generation-with-underscores-in-the-file-name/400079" title="Incorrect attachment markup generation with underscores in the file name - Bug - Discourse Meta" rel="nofollow">Incorrect attachment markup generation with underscores in the file name - Bug - Discourse Meta</a></p> <br><p>主要针对文件名是：<code>_test_file_.txt</code></p> <br><p>如果上传的附件文件名是上面的格式的话，将会导致 discourse 无法完全识别这个文件名，让附件显示不正常。</p> <br><p></p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/9/a/9a5be1e10bd846a7c6dc2a50bac87d4c8d24a74f.jpeg" rel="nofollow"><img alt="2026-05-0110-03-10" height="500" src="https://i-blog.csdnimg.cn/img_convert/fac619401c70e158293d9738b8d10a7c.jpeg" width="504" /></a></p> <br><p></p> <br><p>随后官方给出的解决方案就是，如果文件名中包含有下划线的话，那么第 1 个下划线将会进行自动添加转义符。</p> <br><p>至于这个下划线是在文件名的第 1 个字符，还是文件名的中间，官方的解释是不重要，只要是有下划线，第 1 个下划线就添加。</p> <br><p><a class="has-card" href="https://www.isharkfly.com/t/discourse/1736" title="https://www.isharkfly.com/t/discourse/1736" rel="nofollow"><span class="link-card-box" contenteditable="false"><span class="link-title">https://www.isharkfly.com/t/discourse/1736</span><span class="link-link"><img class="link-link-icon" src="https://csdnimg.cn/release/blog_editor_html/release2.4.6/ckeditor/plugins/CsdnLink/icons/icon-default.png?t=Q239" />https://www.isharkfly.com/t/discourse/1736</span></span></a></p>
