<p>本文将会指导你如何在 Discourse AI 中设置 Google 的 Gemini API key。</p> <br><p>在这之前，你需要有一个 Google Cloud 账号，Gemini AI 不是一个免费的服务，所以你需要在 Google 中设置账号信息等。</p> <br><p>同时，你还需要有 Google Cloud 的管理权限才行。</p> <br><p>一些有关 Google Key 创建的重要内容，可以参考 <a href="https://aistudio.google.com/" rel="nofollow" title="Gemini Developer studio">Gemini Developer studio</a> 页面中的信息。</p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/e/4/e46458cb81295fce9bde6e439a79c2ec33b2dd1c.png" rel="nofollow"><img alt="61423e3471367450d097d5baed34d0231d6ea739_2_690x159" height="159" src="https://i-blog.csdnimg.cn/img_convert/6379c798effbdc2d875f58ec08c7745b.png" width="690" /></a></p> <br><p></p> <br><blockquote> <br> <p></p> <br> <p class="img-center"><img alt=":point_up:" height="72" src="https://i-blog.csdnimg.cn/img_convert/3755eead28743aab29ac67aacd74a8cd.png" width="72" /></p> <br> <p> <strong>NOTE</strong>: 针对 Google Cloud，你需要有管理员权限才可以，否则主要是因为 Key 的权限问题。</p> <br></blockquote> <br><p>哪怕你是通过 Gemini Developer studio 来创建 Key，在 Key 的创建过程中，也还是需要你提供 Google Cloud的项目信息。</p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/6/6/662c4c220c4279d7e420399749bc3fa74c14208c.jpeg" rel="nofollow"><img alt="2025-06-12_10-06-45" height="373" src="https://i-blog.csdnimg.cn/img_convert/464638080bcb71796ac4b220d3395ee7.jpeg" width="690" /></a></p> <br><p></p> <br><p>上面 拉出来的项目都是我们已经在 Google Cloud 中创建的项目。</p> <br><hr /> <br><h3>第一步：设置 Google Cloud 项目</h3> <br><p>设置 Gemini API key 可能需要你的 Google Cloud 管理员权限，同时还需要你创建一个支付账户，这样当你的项目在使用 Gemini API 的时候，Google 将会以云平台账单的形式通过你的支付账户来进行支付。</p> <br><p>当然，你可以通过 Gemini Studio 来创建项目并且连接到项目中。</p> <br><p>但我们发现，直接通过 Google Cloud 的管理员控制台来进行处理更加简单。</p> <br><p>进入管理员控制台： <a href="https://console.cloud.google.com/apis/credentials" rel="nofollow" title="Google Cloud admin for API">Google Cloud admin for API</a> 在进入控制台后，需要确定希望使用 key 的项目。</p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/d/8/d8307545bf2040b26da64ba0b928e8f4da1149d2.png" rel="nofollow"><img alt="14a3bcd1a475c66c83a74c3cfe415feeaa9a0ebe" height="328" src="https://i-blog.csdnimg.cn/img_convert/9abc6ca5a76aa6256e017ac9477687be.png" width="690" /></a></p> <br><p>因为在 Google Cloud 的设计中，你可能会有很多的 Cloud 项目，在这里选一个你希望使用的项目。</p> <br><p></p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/5/e/5ec5cdbac9a0a42a9bd9d518735fc4c6987f0e16.jpeg" rel="nofollow"><img alt="b0b3958294b31ae4d04c87df188461250082612d" height="319" src="https://i-blog.csdnimg.cn/img_convert/f91161c182400e4e1312e6b4312d142c.jpeg" width="690" /></a></p> <br><p>随后就在项目中为项目启用相关的服务就好了。</p> <br><p></p> <br><p>如果你还没有项目的话，你就需要创建一个项目了。</p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/2/1/214b688346bdaa58052ca9cc6a6dbc216ee637c5.png" rel="nofollow"><img alt="7c48b3057b521f63909b6e2ce213d316ad2d5dc4" height="500" src="https://i-blog.csdnimg.cn/img_convert/bba44c5decd47c959f943acf5ad5be93.png" width="569" /></a></p> <br><p>为了管理你的支出，你还需要为你的 Google Cloud 服务创建一个支付账户。</p> <br><p></p> <br><p>然后把你的项目关联进去就好了。</p> <br><hr /> <br><h3>第二步：创建 Gemini API Key</h3> <br><blockquote> <br> <p class="img-center"><img alt=":warning:" height="72" src="https://i-blog.csdnimg.cn/img_convert/449d625e18f835d4b0a65451c2ae1abb.png" width="72" /></p> <br> <p> <strong>IMPORTANT!</strong> 记住需要妥善保管你的这个 Key，请阅读 <a href="https://ai.google.dev/gemini-api/docs/api-key#security" rel="nofollow" title="Keep your API key secure">Keep your API key secure</a> 和 <a href="https://support.google.com/googleapi/answer/6310037" rel="nofollow" title="Best practices for securely using API keys">Best practices for securely using API keys</a> 页面中的内容。</p> <br></blockquote> <br><p>因为你的 key 可能会导致你的账单增加。</p> <br><p>单击页面中的创建一个 key，然后把创建的 key 给保存下来。</p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/5/8/58bb19dea191c952ea57842d7fba154ade7e95e6.png" rel="nofollow"><img alt="48a3d7aff739c933815bc197586c3bfb364e99f1_2_690x198" height="198" src="https://i-blog.csdnimg.cn/img_convert/72b7fdeef386511d7430cfd2dbd5b5b7.png" width="690" /></a></p> <br><p>这个 key 就是我们需要在 Discourse AI 中进行设置需要的。</p> <br><p></p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/e/3/e384f2aeec3e5ceef32cd8ff5b018d55bf643c48.jpeg" rel="nofollow"><img alt="60319530108169db96ec749bcf7cf6d180134274_2_517x305" height="305" src="https://i-blog.csdnimg.cn/img_convert/796b320fb04cc59ccadd55620a8928ca.jpeg" width="517" /></a></p> <br><p>这个 Key 现在应该设置在你的 key 列表中了。</p> <br><p></p> <br><p>单击编辑这个 key，我们需要为这个 key 设置使用权限。</p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/5/5/55f717cd0d8e4ad548ab8ce856dfb8155a428356.png" rel="nofollow"><img alt="b0d62d9e59dde0ccca2e1cb04014da33dc970c5c_2_689x214" height="214" src="https://i-blog.csdnimg.cn/img_convert/34e0a73b45d38f48eec4dd8fd8d3cac6.png" width="689" /></a></p> <br><p>设置这个 key 只能被用于进行 AI 的调用，选择：<br /> “Restrict key”，然后选择 “Generative Language API”.</p> <br><p></p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/0/4/040fc841bb79ddee220afa609f562ff4714f05ba.jpeg" rel="nofollow"><img alt="2025-06-12_10-20-02" height="500" src="https://i-blog.csdnimg.cn/img_convert/24ee260d3bc02fdfd91fe0353d343292.jpeg" width="531" /></a></p> <br><p></p> <br><p>然后进行保存。</p> <br><p>当然，你还可以限制只有你的网站能够调用这个 key。</p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/e/6/e6b4d81aa59135a1e81c298ac053e958d192baa7.jpeg" rel="nofollow"><img alt="2025-06-12_10-21-53" height="499" src="https://i-blog.csdnimg.cn/img_convert/017227f6d5d7c9e72bd9b80d96c8bbd3.jpeg" width="689" /></a></p> <br><p></p> <br><p>这样的话，哪怕是这个 Key 被泄露了，别人也无法通过其他的域名来进行调用，等于这样能够获得双重安全性。</p> <br><p>所以，Google Cloud 的 Key 是可以通过 显示 Key 来多次显示的。</p> <br><p></p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/9/f/9fe14dfde21b11e571b8ec43f5873930fadb65d9.jpeg" rel="nofollow"><img alt="2025-06-12_10-23-49" height="196" src="https://i-blog.csdnimg.cn/img_convert/1864188553659e1c61aaafed33c57429.jpeg" width="690" /></a></p> <br><p></p> <br><p>感觉这个点就比较踏实。</p> <br><p>毕竟 key 太多了，自己都忘记了。</p> <br><hr /> <br><h3>第三步：在 Discourse AI 插件中进行配置</h3> <br><p>现在，你需要在 Discourse 的 AI 插件中进行配置，配置路径为： <code>Admin > Plugins > AI > LLMs</code> 然后选择需要的语言模型进行配置。</p> <br><p></p> <br><p></p> <br><p class="img-center"><a src="https://i-blog.csdnimg.cn/img_convert/80e9309bd510abc887dc0e98ec59d01a.jpeg" width="502" /></a></p> <br><p></p> <br><p>随后可以进行测试，如果显示测试成功，则说明一切正常。</p> <br><p></p> <br><p class="img-center"><a
