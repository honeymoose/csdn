<p>发现一个可用的库： pydiscourse</p> <br><p>这个库可以直接在程序中调用 Discourse 接口。</p> <br><p></p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/7/e/7eb1f7e72ec333ce830814d7a0991dd19d153877.jpeg" rel="nofollow"><img alt="2026-02-20_08-06-02" height="500" src="https://i-blog.csdnimg.cn/img_convert/55154498c3c8d9f8dfae82ed34d2cf5b.jpeg" width="631" /></a></p> <br><p></p> <br><p>我们发现非常有用的地方在用户处理的时候，或者说当网站进行迁移，有很多用户需要添加的时候。</p> <br><p>一个一个的用户添加实在太费劲，可以导出成一个CSV文件，然后调用接口直接读这个CS文件，而一次性把所有用户全部添加到新的系统中。</p> <br><p>添加成功以后的用户的问题是用户名的密码，可能会采用随机密码，或者说某一个随机字符串。</p> <br><p>用户如果需要进行新的网站登录的话，他需要重置自己的密码，当然如果用户的邮箱地址没有问题，它是可以随时进行重置的。</p> <br><p>这个项目的使用场景并不会很高，因为很多时候我们不一定会使用库来调用 Discourse 的API。</p> <br><p>Discourse API 很多时候都已经是 http 写死的，在 Java 项目中，我们就直接定一个工具文件，用于维护连接就可以了。</p> <br><p>只对于一次性的导入导出以及用户管理或者说是一个大规模的数据迁移来讲，API的效率要比人工高得多得多得多。</p> <br><p><a href="https://www.isharkfly.com/t/discourse-python-api/18667" title="https://www.isharkfly.com/t/discourse-python-api/18667" rel="nofollow">https://www.isharkfly.com/t/discourse-python-api/18667</a></p>
