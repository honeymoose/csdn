<p>出现的问题：</p> <br><p>sh ‘pnpm approve-builds --all’ 在 sh ‘pnpm install --no-frozen-lockfile’ 前面的时候第一次构建总是出错。</p> <br><p></p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/0/7/0776a6cdd4258a3fe39aecd18aca07b62815a6e6.jpeg" rel="nofollow"><img alt="2026-05-3109-33-38" height="500" src="https://i-blog.csdnimg.cn/img_convert/20228b056f55c77734b5575fb31da83b.jpeg" width="658" /></a></p> <br><p></p> <br><p>但第二次构建的时候就成功。</p> <br><h3>处理</h3> <br><p>是正常现象：<code>pnpm approve-builds --all</code> 必须在 <strong>已经有 pending/ignored build scripts</strong> 之后才有东西可批准。</p> <br><p>第一次构建时你把它放在 <code>pnpm install --no-frozen-lockfile</code> 前面，它还不知道哪些依赖需要跑 build script，所以批准不到任何包；随后 <code>install</code> 才发现并拦截这些脚本，导致首次构建失败。</p> <br><p>pnpm 文档说明 <code>approve-builds</code> 是批准依赖在安装期间运行脚本，批准项会写入 <code>onlyBuiltDependencies</code>。<code>--all</code> 是批准所有 pending builds。</p> <br><p>推荐顺序：</p> <br><p></p> <br><pre><code>pnpm install --no-frozen-lockfile<br>pnpm approve-builds --all<br>pnpm rebuild<br>pnpm build<br></code></pre> <br><p>核心原因：<code>approve-builds</code> 不是“预先允许所有未来依赖”，而是“批准当前已被 pnpm 发现、正在等待批准的 build scripts”。</p> <br><p>如果你的 Agent 是全新的（Docker/K8s Pod 每次重建），可以：</p> <br><p></p> <br><pre><code>pipeline {<br>    agent any<br><br>    stages {<br>        stage('Install') {<br>            steps {<br>                sh '''<br>                    corepack enable<br><br>                    # 第一次安装，生成 pending builds<br>                    pnpm install --no-frozen-lockfile || true<br><br>                    # 自动批准<br>                    pnpm approve-builds --all<br><br>                    # 重新执行安装<br>                    pnpm install --no-frozen-lockfile<br>                '''<br>            }<br>        }<br><br>        stage('Build') {<br>            steps {<br>                sh 'pnpm build'<br>            }<br>        }<br>    }<br>}<br></code></pre> <br><p>最终使用的脚本是添加一个 Stage</p> <br><p></p> <br><pre><code>        stage("Install") {<br>            steps {<br>                sh '''<br>                    pnpm install --no-frozen-lockfile || true<br>                    pnpm approve-builds --all<br>                    pnpm install --no-frozen-lockfile<br>                '''<br>            }<br>        }<br></code></pre> <br><p></p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/7/a/7a98b869132059e2f30a705f3718ef1694909b32.jpeg" rel="nofollow"><img alt="2026-05-3109-37-58" height="335" src="https://i-blog.csdnimg.cn/img_convert/724673add3e7f46a84893203f0e8141b.jpeg" width="690" /></a></p> <br><p><a href="https://www.isharkfly.com/t/pnpm-approve-builds-all-jenkins/10046" title="Pnpm approve-builds --all Jenkins 第一次构建总是出错 - DevOps - iSharkFly" rel="nofollow">Pnpm approve-builds --all Jenkins 第一次构建总是出错 - DevOps - iSharkFly</a></p> <br><p></p> <br><p></p>
