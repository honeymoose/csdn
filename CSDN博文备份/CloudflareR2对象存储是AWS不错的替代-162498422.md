<p>最早开始接触 AWS 是从S3对象存储开始的，这个已经都是快 20年了。</p> <br><p>随着 AWS 的使用越来越深入，会发现 AWS 的很多配置设置的过于复杂和过于琐碎。</p> <br><p>但是 AWS 非常擅长于宣传，他们给市场的说法是，AWS只是像乐高一样提供组件。</p> <br><p>但是对很多公司来说，他们真正用不到AWS过多的功能，可能用的比较多的是存储、数据库以及一些计算能力。</p> <br><p>就现在 AWS 配置来说，随便一个小的改动，可能会涉及到好多组件，对 AWS 自己来说，给人的感觉是很多东西他们自己内部其实都不怎么能通畅通信，很多团队只负责自己一小块，至于怎么跟其他团队去进行更好的有机整合，以及为客户提供更多的便利，他们不管。</p> <br><p>正是基于这种考虑，最近在尝试使用 Cloudflare 的 R2 对象存储。</p> <br><p></p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/1/4/149a33dc41e7d4ee3aa494cdcba2e1715f5816f7.jpeg" rel="nofollow"><img alt="2026-07-01_08-46-21" height="448" src="https://i-blog.csdnimg.cn/img_convert/acb3b63023b174ef207ee6f57a9af94c.jpeg" width="690" /></a></p> <br><p></p> <br><p>主要的还是因为最近在使用 Cloudflare 的无服服务计算。</p> <br><p>只需要把代码推上去就可以提供服务，给人的感觉是非常方便，并不需要像AWS一样设置一大堆乱七八糟的，花了一两天的时间，还不知道自己要干什么。</p> <br><p>而且对初始团队来讲，它的部署和解构非常迅速，价格也比 AWS 亲民，提供了足够的免费试用量。</p> <br><p></p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/d/a/dad0fe67879afc2b1e9d8ba4da2261a6de7261bd.jpeg" rel="nofollow"><img alt="2026-07-01_08-50-57" height="491" src="https://i-blog.csdnimg.cn/img_convert/0e3b74e8a16f37c5fa5bb213ec8f3796.jpeg" width="690" /></a></p> <br><p></p> <br><p>正是因为这种考虑，于是尝试使用一下 R2 对象存储。</p> <br><p>至少从界面来讲，把数据从 AWS 搬到 R 上面还是非常方便的，直接使用他们的数据合并工具，就可以把对象拉过来。</p> <br><p>相对功能来讲，比如说在界面中移动文件夹，R2 没有提供这个操作，UI 上面的功能比 S3 要小一些。</p> <br><p>对于应用来讲，如果是纯程序操作，并且需要使用 CDN 的话，我觉得是可以使用 R2 做一些替代性尝试。</p> <br><p><a href="https://www.isharkfly.com/t/cloudflare-r2-aws/2317" title="Cloudflare R2 对象存储是 AWS 不错的替代 - DevOps - iSharkFly" rel="nofollow">Cloudflare R2 对象存储是 AWS 不错的替代 - DevOps - iSharkFly</a></p>
