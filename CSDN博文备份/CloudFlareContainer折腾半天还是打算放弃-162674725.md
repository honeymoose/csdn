<p>本想着使用 Cloudflare 的容器做无服务应用部署。</p> <br><p>折腾了10多个小时，还是无法验证部署服务的启动端口。</p> <br><p>容器的部署没有想象中的那么简单，不是说在一个服务器上直接装一个 Docker 的引擎，就可以直接部署上去，没有问题。</p> <br><p>无服务的容器部署相对来讲，可能出现的问题很多。</p> <br><h3>验证端口</h3> <br><p>同样的服务部署在 Docker 容器的纯服务器上就没有问题。</p> <br><p></p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/3/5/35eb10e7159e14babdd94bcfdcd96e8de8d11d9c.jpeg" rel="nofollow"><img alt="2026-07-07_10-05-48" height="344" src="https://i-blog.csdnimg.cn/img_convert/13abf7356192809ced405839a8ab68f8.jpeg" width="690" /></a></p> <br><p></p> <br><p>更糟糕的是服务一直起不来，查看日志，服务已经起来了，端口也是对的，地址绑定也没问题，但是容器的连接校验就是不对。</p> <br><p>下面的日志显示的是 Spring Boot 的日志输出，因为他们的日志读取模式和普通的日志读取模式不一样， 查找起来很不直观。</p> <br><h3>冷启动</h3> <br><p>同时还有另外一个问题，就是容器的冷启动问题。</p> <br><p>因为系统的访问量在短期之内不会有那么多的访问量，因此容器的在线时间会被设置在10分钟左右。</p> <br><p>就是因为这个设置会导致容器的冷启动，意思就是说在10分钟以内容器的应用没有访问的话，容器会被停止运行。</p> <br><p></p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/1/8/18ebb2a7457edcd02d7cd1cef2cdb548dc7b605e.jpeg" rel="nofollow"><img alt="2026-07-07_10-07-32" height="372" src="https://i-blog.csdnimg.cn/img_convert/561a912b41e1a6e1a07534db3202863d.jpeg" width="690" /></a></p> <br><p></p> <br><p>比如说现在容器的运行状态就是Ready，不是 active。</p> <br><p>作为 SpringBoot 的应用程序来讲，下一次接受访问的时候，容器会被启动，启动的时间会要好几分钟。</p> <br><p>据说这样做的目的是为了节约使用费用，因为当没有访问的时候容器被停止，被停止的容器是不会被计费的。</p> <br><p>我感觉这个调试上来就是很麻烦。</p> <br><h3>容器更新</h3> <br><p>在开发的过程中，我们会对同一个 Darker 的 image 进行多次同一版本多次推送。</p> <br><p>因为程序代码的变化没有，所以在部署到容器上面去的时候，容器会默认为这次没有变化，然后整个容器不更新。</p> <br><p>这个就给调试带来很大的麻烦，有时候甚至都不知道容器部署上去，到底有没有更新。</p> <br><p></p> <br><p></p> <br><p class="img-center"><a href="https://cdn.isharkfly.com/com-isharkfly-www/discourse-uploads/original/3X/2/a/2af02186b2c390a779759bad277c074e364421e1.jpeg" rel="nofollow"><img alt="2026-07-07_10-39-22" height="253" src="https://i-blog.csdnimg.cn/img_convert/dac3a478dd737ff9e7e9f4093c2cf4f9.jpeg" width="690" /></a></p> <br><p></p> <br><p>比如说我们这里已经推送过多次了，但是在版本这里还是显示 V1 的版本，没有办法检测更新的状态。</p>
